# 开发任务拆解书

下面这份“开发任务拆解书”是按你已经确认的系统设计来写的，目标只有一个：**让开发人员按最少功能、最少耦合、最少代码量，顺序完成整条功能链路**。

我先给出总原则，再按任务顺序拆开。每一步都包含：

- 要做什么
- 最小实现要求
- 关键代码骨架
- 验收清单
- 验收流程

------

# 一、开发前的硬规则

这版 MVP 严格遵守下面 6 条，不要提前做“增强版”：

1. **后端先行，前端后接。**
   先用 Swagger 把整条链路跑通，再写前端页面。
2. **先接 mock，再接真实模型。**
   抽取模型和问答模型都先做 mock 适配，避免开发被外部 API 阻塞。
3. **不引入多余中间层。**
   这版采用：
   **router -> service -> SQLAlchemy / Neo4j / LLM client**
   不单独拆 repository 层，不引入 CQRS，不引入消息队列。
4. **不引入重型基础设施。**
   这版不做：
   - Celery / Redis
   - Alembic
   - WebSocket
   - 登录鉴权
   - 动态 Neo4j Label / 动态关系类型
   - 复杂图可视化
5. **所有模型调用都走统一接口，但允许 provider / 模型适配。**
   抽取模型、Cypher 生成、答案生成，都复用同一个 `llm_client` 抽象；但不得假设所有 OpenAI-compatible provider 共享同一消息结构、同一 SDK 行为和同一传输策略。
6. **图谱变更后的规则要简单。**
   只要三元组或 schema 被改过，就把项目状态从 `imported` 打回 `initialized`。
   这意味着：**图谱已过期，后续应优先用 `rebuild` 重建，而不是指望增量同步删除。**

------

# backend/app/core/database.py
from sqlalchemy import create_engine, event
from sqlalchemy.orm import declarative_base, sessionmaker
from app.core.settings import settings

engine = create_engine(
    settings.sqlite_url,
    connect_args={"check_same_thread": False},
)

@event.listens_for(engine, "connect")
def enable_sqlite_fk(dbapi_connection, connection_record):
    cursor = dbapi_connection.cursor()
    cursor.execute("PRAGMA foreign_keys=ON")
    cursor.close()

SessionLocal = sessionmaker(
    bind=engine,
    autoflush=False,
    autocommit=False,
    expire_on_commit=False,
)

Base = declarative_base()

def get_db():
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()
```

# T06 抽取模型 client 与异步批处理

## 目标

实现“启动抽取任务 -> 后台逐条请求模型 -> 保存原始响应 -> 更新进度”，并把真实 provider 接入从“固定 HTTP 请求”升级为“统一接口 + provider / 模型兼容适配”。

## 任务内容

## 1. 统一模型 client 接口

这一阶段要统一的是**服务内部调用接口**，而不是要求所有真实 provider 都直接接受同一份 HTTP 请求体。

系统内部统一调用语义仍然是：

```json
{
  "model": "xxx",
  "instruction": "...",
  "input": "..."
}
```

但真实 provider 侧必须允许以下兼容策略：

- 抽取场景默认把 `instruction + input_text` 合并为**单条 user 消息**
- `system` 消息不是强制方案，只能作为可选 provider 策略
- 允许使用官方 / 兼容 SDK，不要求必须手写 `httpx.post`
- 允许使用 `stream=True` 再由客户端聚合最终输出
- 允许按 provider / 模型注入兼容参数，例如 `temperature`、`top_p`、`thinking_budget`、`max_tokens`

### 当前推荐真实验收组合（截至 2026-03-18）

- `SiliconFlow + deepseek-ai/DeepSeek-V3.2`
- `deepseek-ai/DeepSeek-R1` 不再作为结构化抽取默认验收模型

### 抽取模型期望响应格式

进入解析层之前，`llm_client` 应把真实 provider 的返回标准化为：

```json
{
  "output": "[{"subject":"...","subject_type":"...","predicate":"...","object":"...","object_type":"..."}]"
}
```

### 问答模型期望响应格式

同样统一为：

```json
{
  "output": "MATCH ... RETURN ..."
}
```

或

```json
{
  "output": "最终答案文本"
}
```

## 2. client 骨架

推荐把 `llm_client.py` 改成“Mock + ProviderAdapter + 统一错误诊断”的结构。

```python
# backend/app/services/llm_client.py
import json
from dataclasses import dataclass
from typing import Any
from urllib.parse import urlparse

from openai import OpenAI


@dataclass
class LLMCallError(Exception):
    error_code: str
    message: str
    exception_type: str
    request_url: str | None = None
    model_name: str | None = None
    status_code: int | None = None
    response_excerpt: str | None = None

    def to_dict(self) -> dict[str, Any]:
        return {
            "error_code": self.error_code,
            "message": self.message,
            "exception_type": self.exception_type,
            "request_url": self.request_url,
            "model_name": self.model_name,
            "status_code": self.status_code,
            "response_excerpt": self.response_excerpt,
        }


def build_extract_user_content(instruction: str, input_text: str) -> str:
    return f"{instruction.strip()}

输入文本：
{input_text.strip()}".strip()


def build_provider_options(base_url: str, model_name: str, instruction: str) -> dict[str, Any]:
    options: dict[str, Any] = {}
    host = urlparse(base_url).netloc.lower()
    model_lower = model_name.lower()
    instruction_lower = instruction.lower()

    if "siliconflow" in host and ("deepseek-r1" in model_lower or "deepseek-v3.2" in model_lower):
        options["temperature"] = 0.6
        options["top_p"] = 0.95
        options["extra_body"] = {"thinking_budget": 128}

    if any(key in instruction_lower for key in ["json", "三元组", "cypher"]):
        options["max_tokens"] = 256

    return options


class ProviderCompatibleClient:
    def call(self, *, base_url: str, api_key: str, model_name: str, instruction: str, input_text: str, timeout_seconds: int) -> str:
        client = OpenAI(base_url=base_url, api_key=api_key, timeout=timeout_seconds)
        messages = [{"role": "user", "content": build_extract_user_content(instruction, input_text)}]
        options = build_provider_options(base_url, model_name, instruction)

        try:
            stream = client.chat.completions.create(
                model=model_name,
                messages=messages,
                stream=True,
                **options,
            )
            chunks: list[str] = []
            for event in stream:
                delta = event.choices[0].delta.content or ""
                if delta:
                    chunks.append(delta)
            return json.dumps({"output": "".join(chunks)}, ensure_ascii=False)
        except Exception as e:
            raise LLMCallError(
                error_code="MODEL_PROVIDER_COMPAT_ERROR",
                message=str(e),
                exception_type=type(e).__name__,
                request_url=base_url,
                model_name=model_name,
            )


class MockCompatibleClient:
    def call(self, *, base_url: str, api_key: str, model_name: str, instruction: str, input_text: str, timeout_seconds: int) -> str:
        if "cypher" in instruction.lower():
            return json.dumps({"output": "MATCH (n:KGEntity) WHERE n.project_id = $project_id RETURN n.name AS name LIMIT 5"}, ensure_ascii=False)
        if "答案" in instruction or "answer" in instruction.lower():
            return json.dumps({"output": "这是基于查询结果生成的模拟答案。"}, ensure_ascii=False)
        return json.dumps({"output": "[]"}, ensure_ascii=False)


def get_llm_client(base_url: str):
    if base_url.startswith("mock://"):
        return MockCompatibleClient()
    return ProviderCompatibleClient()

```python
# backend/app/services/extraction_service.py
import json
from app.core.database import SessionLocal
from app.models.sqlite_models import ImportBatch, SourceRecord, Project, ModelConfig
from app.services.llm_client import get_llm_client, LLMCallError
from app.utils.time_utils import utcnow_str


def run_batch_extract_task(batch_id: str):
    db = SessionLocal()
    try:
        batch = db.get(ImportBatch, batch_id)
        project = db.get(Project, batch.project_id)
        config = db.get(ModelConfig, project.extract_config_id)

        sources = (
            db.query(SourceRecord)
            .filter(SourceRecord.batch_id == batch_id)
            .filter(SourceRecord.request_status.in_(["pending", "failed"]))
            .order_by(SourceRecord.line_no.asc())
            .all()
        )

        client = get_llm_client(config.base_url)

        for source in sources:
            try:
                source.request_status = "running"
                source.updated_at = utcnow_str()
                db.commit()

                request_payload = {
                    "model": config.model_name,
                    "instruction": batch.instruction,
                    "input": source.input_text,
                }
                source.request_payload = json.dumps(request_payload, ensure_ascii=False)

                raw_response = client.call(
                    base_url=config.base_url,
                    api_key=config.api_key,
                    model_name=config.model_name,
                    instruction=batch.instruction or "",
                    input_text=source.input_text,
                    timeout_seconds=config.timeout_seconds,
                )

                source.raw_response = raw_response
                source.request_status = "success"
                source.error_message = None
                source.updated_at = utcnow_str()
                db.commit()

                parse_service.reparse_source(db, source.id)

            except LLMCallError as e:
                source.request_status = "failed"
                source.error_message = json.dumps(e.to_dict(), ensure_ascii=False)
                source.retry_count += 1
                source.updated_at = utcnow_str()
                db.commit()
            except Exception as e:
                source.request_status = "failed"
                source.error_message = json.dumps({
                    "error_code": "MODEL_UNKNOWN_ERROR",
                    "message": str(e),
                    "exception_type": type(e).__name__,
                    "request_url": config.base_url,
                    "model_name": config.model_name,
                }, ensure_ascii=False)
                source.retry_count += 1
                source.updated_at = utcnow_str()
                db.commit()

        refresh_batch_status(db, batch_id)
    finally:
        db.close()
```

# T07 原始响应修正、解析、三元组编辑

## 目标

实现这条关键链路：

**原始响应 -> 解析 output -> 生成三元组 -> 允许人工修正 -> 再解析 -> 三元组增删改**

## 任务内容

## 1. 解析规则

必须支持：

- 从原始响应 JSON 中取 `output`
- `output` 必须是字符串
- 去掉 Markdown 代码块
- 去掉首尾空白
- 解析成 JSON 数组
- 每个数组项必须含 5 个字段：
  - `subject`
  - `subject_type`

  - `object`
  - `object_type`

若任一字段缺失，**整条 source 解析失败**。

## 2. 解析工具

```python
# backend/app/utils/json_cleaner.py
import json
import re

REQUIRED_TRIPLE_KEYS = ["subject", "subject_type", "predicate", "object", "object_type"]

class ParseError(Exception):
    pass

def clean_output_text(output_text: str) -> str:
    text = output_text.strip()
    text = re.sub(r"^```(?:json)?\s*", "", text, flags=re.IGNORECASE)
    text = re.sub(r"\s*```$", "", text)
    return text.strip()

def extract_output_string(raw_response: str) -> str:
    try:
        obj = json.loads(raw_response)
    except Exception:
        raise ParseError("RAW_RESPONSE_NOT_JSON")

    if "output" not in obj:
        raise ParseError("OUTPUT_FIELD_MISSING")

    if not isinstance(obj["output"], str):
        raise ParseError("OUTPUT_TYPE_INVALID")

    return clean_output_text(obj["output"])

def parse_output_to_triples(raw_response: str) -> list[dict]:
    output_text = extract_output_string(raw_response)

    try:
        data = json.loads(output_text)
    except Exception:
        raise ParseError("INVALID_JSON_OUTPUT")

    if not isinstance(data, list):
        raise ParseError("OUTPUT_NOT_ARRAY")

    triples = []
    for item in data:
        if not isinstance(item, dict):
            raise ParseError("TRIPLE_ITEM_NOT_OBJECT")
        for key in REQUIRED_TRIPLE_KEYS:
            if key not in item:
                raise ParseError(f"TRIPLE_FIELD_MISSING:{key}")

        triple = {
            "subject": str(item["subject"]).strip(),
            "subject_type": str(item["subject_type"]).strip(),
            "predicate": str(item["predicate"]).strip(),
            "object": str(item["object"]).strip(),
            "object_type": str(item["object_type"]).strip(),
        }
        if not all(triple.values()):
            raise ParseError("TRIPLE_FIELD_EMPTY")
        triples.append(triple)

    return triples, output_text
```

# backend/app/services/cypher_validator.py
import re

FORBIDDEN_KEYWORDS = [
    "CREATE", "MERGE", "DELETE", "DETACH", "SET", "REMOVE",
    "CALL", "LOAD CSV", "DROP", "FOREACH", "APOC"
]

def validate_readonly_cypher(cypher: str) -> tuple[bool, str | None]:
    if not cypher or not cypher.strip():
        return False, "EMPTY_CYPHER"

    text = cypher.strip()
    upper = text.upper()

    if ";" in text:
        return False, "MULTI_STATEMENT_NOT_ALLOWED"

    for kw in FORBIDDEN_KEYWORDS:
        if re.search(rf"\b{re.escape(kw)}\b", upper):
            return False, f"FORBIDDEN_KEYWORD:{kw}"

    if "PROJECT_ID" not in upper:
        return False, "PROJECT_SCOPE_MISSING"

    if "MATCH" not in upper and "OPTIONAL MATCH" not in upper:
        return False, "NO_MATCH_CLAUSE"

    return True, None
```

    generated_cypher = extract_output_string(raw_cypher_resp)
    is_valid, reason = validate_readonly_cypher(generated_cypher)

    if not is_valid:
        # 记录 qa_log
        return {
            "status": "cypher_invalid",
            "generated_cypher": generated_cypher,
            "query_result": None,
            "answer": None,
        }

    driver = get_driver()
    with driver.session() as session:
        result = session.run(generated_cypher, project_id=project_id)
        rows = [dict(r) for r in result]

    if not rows:
        answer = "当前知识图谱中未检索到相关信息。"
        # 写 qa_log
        return {
            "status": "no_result",
            "generated_cypher": generated_cypher,
            "query_result": [],
            "answer": answer,
        }

    answer_instruction = build_answer_instruction()
    answer_input = f"问题：{question}\nCypher：{generated_cypher}\n查询结果：{json.dumps(rows, ensure_ascii=False)}"
    raw_answer_resp = client.call(
        base_url=qa_config.base_url,
        api_key=qa_config.api_key,
        model_name=qa_config.model_name,
        instruction=answer_instruction,
        input_text=answer_input,
        timeout_seconds=qa_config.timeout_seconds,
    )
    answer = extract_output_string(raw_answer_resp)

    # 写 qa_log
    return {
        "status": "success",
        "generated_cypher": generated_cypher,
        "query_result": rows,
        "answer": answer,
    }
```

# 三、必须补充给开发人员的实现注意事项

## 1. 这版不要做 repository 层

直接：

- route 收参
- service 处理业务
- service 里直接用 SQLAlchemy session

这样代码量最小，后续也容易改。

------

## 2. BackgroundTasks 里必须新建 DB session

不要把请求里的 `db` 传进后台任务继续用，否则很容易出线程和生命周期问题。

------

## 3. 所有“改数据”的动作都要打脏项目状态

这些动作发生时，把项目状态从 `imported` 改回 `initialized`：

- 修改 raw_response
- reparse 成功
- 手工新增三元组
- 编辑三元组
- 删除三元组
- 改实体类型名
- 改关系类型名

------

## 4. 导入图谱时优先相信 SQLite，Neo4j 只是投影

这版不要尝试做 SQLite 与 Neo4j 的双向同步。
**SQLite 是业务真相源，Neo4j 是查询投影。**

------

## 5. 前端不要先做美化

先让下面 4 个按钮都可用：

- 上传
- 抽取
- 刷新 schema
- 导入 / 重建图谱
- 问答

样式最后做。
