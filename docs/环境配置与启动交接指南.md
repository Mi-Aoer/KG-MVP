# 环境配置与启动交接指南

## 1. 项目运行结构

本项目是本地单机运行的前后端分离系统：

- 后端：FastAPI，负责业务接口、SQLite 元数据、Neo4j 图数据库写入、LLM 抽取调用。
- 前端：React + Vite，负责页面交互。
- 元数据库：SQLite，默认文件为 `data/app.db`，启动后自动创建。
- 图数据库：Neo4j 5.x，通过 Bolt 协议连接。
- 大模型抽取：配置保存在业务库中，不固定写在 `.env`。

推荐启动顺序：

```bash
Neo4j -> 后端 -> 前端
```

## 2. 必备环境

| 组件 | 要求 | 说明 |
| --- | --- | --- |
| Python | 3.11 | 后端启动脚本优先查找 `python3.11` |
| Node.js | 20+ | 前端 Vite / React 运行环境 |
| npm | 10+ | 随 Node.js 安装即可 |
| Neo4j | 5.x | 本地图数据库，需开启 Browser 和 Bolt |
| Git | 可选但推荐 | 用于拉取或同步代码 |

建议端口：

| 服务 | 默认地址 | 用途 |
| --- | --- | --- |
| 后端 | `http://127.0.0.1:8000` | FastAPI 接口 |
| Swagger | `http://127.0.0.1:8000/docs` | 接口文档 |
| 健康检查 | `http://127.0.0.1:8000/health` | 后端存活检查 |
| 前端 | `http://127.0.0.1:5173` | Vite 开发服务 |
| Neo4j Browser | `http://localhost:7474` | 图数据库管理界面 |
| Neo4j Bolt | `bolt://localhost:7687` | 后端连接 Neo4j |

如果端口已被占用，需要修改对应启动参数或配置。

## 3. 获取项目代码

将项目目录完整复制或拉取到新电脑：

```bash
git clone https://github.com/Mi-Aoer/KG-MVP.git
cd KG-MVP
```

## 4. 配置 Neo4j

1. 安装 Neo4j 5.x，推荐使用 Neo4j Desktop 或本地服务版。
2. 创建或启动一个本地数据库。
3. 确认 Browser 可以访问：

```text
http://localhost:7474
```

4. 确认 Bolt 地址为：

```text
bolt://localhost:7687
```

5. 确认用户名和密码与后端配置一致。

项目默认配置如下：

```env
NEO4J_URI=bolt://localhost:7687
NEO4J_USER=neo4j
NEO4J_PASSWORD=xpsdd520
```

如果新电脑上的 Neo4j 密码不同，修改 `backend/.env` 中的 `NEO4J_PASSWORD` 即可。

## 5. 后端配置与启动

推荐使用项目自带稳定启动脚本。该脚本会自动创建虚拟环境、安装依赖、复制 `.env.example` 为 `.env`，并启动后端。

在项目根目录执行：

```bash
bash backend/scripts/start_backend_stable.sh
```

脚本默认行为：

- 虚拟环境路径：`$HOME/.venvs/kg-mvp-backend-py311`
- 后端地址：`http://0.0.0.0:8000`
- 自动读取配置文件：`backend/.env`
- 依赖文件变化后自动重新安装：`backend/requirements.txt`

如果需要指定端口或监听地址：

```bash
KGQA_BACKEND_VENV=$HOME/.venvs/kg-mvp-backend-py311 \
HOST=127.0.0.1 \
PORT=8000 \
RELOAD=1 \
bash backend/scripts/start_backend_stable.sh
```

后端启动成功后，打开：

```text
http://127.0.0.1:8000/health
http://127.0.0.1:8000/docs
```

### 后端 `.env` 配置

首次启动脚本会自动生成 `backend/.env`。也可以手动复制：

```bash
cp backend/.env.example backend/.env
```

推荐初始配置：

```env
APP_NAME=kg-mvp
APP_ENV=dev
DEBUG=true

SQLITE_URL=sqlite:///../data/app.db

NEO4J_URI=bolt://localhost:7687
NEO4J_USER=neo4j
NEO4J_PASSWORD=xpsdd520

CORS_ORIGINS=http://localhost:5173,http://127.0.0.1:5173
```

配置说明：

| 变量 | 说明 |
| --- | --- |
| `SQLITE_URL` | SQLite 数据库路径，默认写入 `data/app.db` |
| `NEO4J_URI` | Neo4j Bolt 连接地址 |
| `NEO4J_USER` | Neo4j 用户名 |
| `NEO4J_PASSWORD` | Neo4j 密码 |
| `CORS_ORIGINS` | 允许访问后端的前端地址 |

### 后端手动启动方式

如果不使用稳定启动脚本，可以手动执行：

```bash
cd backend
python3.11 -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
pip install -r requirements.txt
cp .env.example .env

```

```powershell
cd backend
py -3.11 -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
pip install -r requirements.txt
```

## 6. 前端配置与启动

打开新的终端窗口，进入前端目录：

```bash
cd frontend
npm ci
npm run dev
```

如果 `npm ci` 因锁文件或 npm 版本问题失败，可以改用：

```bash
npm install
npm run dev
```

前端默认请求：

```text
http://127.0.0.1:8000/api
```

如果后端端口不是 8000，启动前端时指定：

```bash
VITE_API_BASE_URL=http://127.0.0.1:8001/api npm run dev
```

Windows PowerShell 可参考：

```powershell
$env:VITE_API_BASE_URL="http://127.0.0.1:8001/api"
npm run dev
```

前端启动成功后访问：

```text
http://127.0.0.1:5173
```

## 7. 大模型抽取配置

大模型配置不写在 `backend/.env` 中，而是在系统页面或接口中创建抽取配置。

无 API Key 时，建议先用 mock 配置跑通主流程：

```text
base_url=mock://extract
model_name=mock
api_key=任意非空字符串
```

如果在实验室服务器（10.108.119.200）上运行大模型，需要先在主机的终端上配置端口转发：
ssh -N -L 11434:127.0.0.1:11434 lqx@10.108.119.200
这样可以把主机发出的大模型访问请求转发到服务器上

## 9. 首次启动验收流程

按以下顺序验证：

1. 启动 Neo4j，并确认 `http://localhost:7474` 可打开。
2. 启动后端，并确认 `http://127.0.0.1:8000/health` 返回正常。
3. 启动前端，并打开 `http://127.0.0.1:5173`。
4. 在系统中创建抽取配置，优先使用 `mock://extract`。
5. 创建项目。
6. 导入 `data/军事新闻实体关系抽取_前10条_系统导入输入.txt`。
7. 执行三元组抽取，检查结果列表。
8. 刷新或维护 schema。
9. 执行图谱初始化和导入，确认 Neo4j 中出现节点与关系。
10. 如需真实验收，再切换 SiliconFlow + DeepSeek-V3.2 配置。

## 10. 常见问题

### 后端启动失败：找不到 Python

确认已安装 Python 3.11：

```bash
python3.11 --version
```

如果只有 `python3`，稳定启动脚本会尝试使用 `python3`，但仍建议安装 Python 3.11。

### 后端依赖安装失败

先确认虚拟环境和 pip 正常：

```bash
python3.11 -m venv /tmp/kg-test-venv
/tmp/kg-test-venv/bin/python -m pip install --upgrade pip
```

如果网络较慢，可切换稳定的 Python 包镜像源后重试。

### 前端启动失败：Node 版本过低

确认版本：

```bash
node -v
npm -v
```

Node.js 建议使用 20 或更高版本。

### 前端请求失败或跨域

检查后端是否启动：

```text
http://127.0.0.1:8000/health
```

检查 `backend/.env`：

```env
CORS_ORIGINS=http://localhost:5173,http://127.0.0.1:5173
```

如果前端端口不是 5173，把实际地址加入 `CORS_ORIGINS`，然后重启后端。

### Neo4j 导入失败

优先检查三项：

- Neo4j Browser 能否打开：`http://localhost:7474`
- Bolt 地址是否正确：`bolt://localhost:7687`
- `backend/.env` 中用户名和密码是否与 Neo4j 一致

修改 `.env` 后需要重启后端。

### mock 抽取返回空数组

mock 逻辑依赖输入文本与内置 JSON 数据匹配。请优先使用：

```text
data/军事新闻实体关系抽取_前10条_系统导入输入.txt
```

如果自行准备文本，未命中 mock 数据时会返回空结果。

### 端口被占用

后端改端口：

```bash
PORT=8001 bash backend/scripts/start_backend_stable.sh
```

前端改后端 API 地址：

```bash
VITE_API_BASE_URL=http://127.0.0.1:8001/api npm run dev
```

### 需要清空本地数据重新演示

停止后端后，可以删除 SQLite 文件：

```bash
rm -f data/app.db
```

Neo4j 中的数据建议通过系统删除项目、图谱清理接口或 Neo4j Browser 手动清理。删除前请确认没有需要保留的数据。

## 12. Windows 实测补充

以下内容来自 Windows 11 + PowerShell + Git Bash 环境的实际配置过程，适合新电脑交接时快速排查。

### Python 版本不符合要求

问题表现：

- `python --version` 显示为 Python 3.13。
- `py -3.11 --version` 提示找不到 Python 3.11。
- 稳定启动脚本需要 Python 3.11 创建后端虚拟环境。

解决方式：

```powershell
winget install --id Python.Python.3.11 --source winget --accept-package-agreements --accept-source-agreements --silent
py -3.11 --version
```

确认能输出 Python 3.11.x 后，再启动后端。

### 稳定启动脚本在 Windows + Git Bash 下的兼容问题

问题表现：

- Git Bash 中可能优先找到 WindowsApps 的 `python3` 占位命令，而不是可用的 Python 3.11。
- Windows 虚拟环境目录是 `Scripts/python.exe`、`Scripts/uvicorn.exe`，不是 Linux/macOS 下的 `bin/python`、`bin/uvicorn`。
- Windows 下直接执行 `pip.exe install --upgrade pip` 可能报错，建议使用 `python -m pip`。

处理方式：

- 启动脚本应优先识别 `py -3.11`。
- 虚拟环境命令应同时兼容 `bin/` 和 `Scripts/`。
- 安装依赖时使用：

```bash
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

本项目的 `backend/scripts/start_backend_stable.sh` 已按上述方式补充 Windows 兼容处理。

### Git Bash 不在 PATH

问题表现：

```powershell
bash --version
```

提示找不到 `bash`，但安装 Git for Windows 后通常实际存在：

```text
C:\Program Files\Git\bin\bash.exe
```

可以用完整路径启动后端脚本：

```powershell
& "C:\Program Files\Git\bin\bash.exe" -lc "HOST=127.0.0.1 PORT=8000 RELOAD=1 bash backend/scripts/start_backend_stable.sh"
```

### 代理配置

如果下载 Python 包、npm 包或访问 GitHub 失败，可临时使用本机代理。例如代理端口为 `127.0.0.1:7897`：

```powershell
$env:http_proxy="http://127.0.0.1:7897"
$env:https_proxy="http://127.0.0.1:7897"
```

Git Bash 中：

```bash
export http_proxy=http://127.0.0.1:7897
export https_proxy=http://127.0.0.1:7897
```

如需给 Git 单独设置代理：

```powershell
git config --global http.proxy http://127.0.0.1:7897
git config --global https.proxy http://127.0.0.1:7897
```

如端口变化或不再需要代理，可清除：

```powershell
git config --global --unset http.proxy
git config --global --unset https.proxy
```

### Neo4j 5.x 本地 zip 安装方式

如果包管理器中没有直接的 Neo4j 5.x 服务端包，可以使用官方 zip 包安装。示例版本：

```powershell
$version="5.26.26"
$url="https://dist.neo4j.org/neo4j-community-$version-windows.zip"
$installRoot="$env:USERPROFILE\Apps\Neo4j"
$zipPath="$env:TEMP\neo4j-community-$version-windows.zip"
New-Item -ItemType Directory -Force -Path $installRoot | Out-Null
Invoke-WebRequest -Uri $url -OutFile $zipPath
Expand-Archive -LiteralPath $zipPath -DestinationPath $installRoot -Force
```

设置项目默认密码：

```powershell
$neo4jHome="$env:USERPROFILE\Apps\Neo4j\neo4j-community-5.26.26"
& "$neo4jHome\bin\neo4j-admin.bat" dbms set-initial-password xpsdd520
```

启动 Neo4j：

```powershell
Start-Process -FilePath "$neo4jHome\bin\neo4j.bat" -ArgumentList "console" -WorkingDirectory $neo4jHome -WindowStyle Hidden
```

确认端口：

```powershell
Test-NetConnection localhost -Port 7474
Test-NetConnection localhost -Port 7687
```

浏览器访问：

```text
http://localhost:7474
```

默认登录信息：

```text
user=neo4j
password=xpsdd520
```

### npm audit 提示

`npm ci` 后可能出现依赖安全审计提示，例如 moderate/high vulnerabilities。这类提示不一定影响本地开发启动。交接演示阶段可先记录，后续升级依赖前应单独评估兼容性，不建议直接无脑执行 `npm audit fix --force`。

## 13. 交接检查清单

交接给别人前，建议确认：

- Neo4j 安装方式、用户名、密码已说明。
- `backend/.env.example` 可用，真实 `.env` 不提交。
- `frontend/package-lock.json` 已保留，便于 `npm ci`。
- `data/` 中演示文本仍存在。
- 新电脑上按本文档可以访问前端、Swagger、Neo4j Browser。
- 至少用 mock 配置跑通过一次“导入文本 -> 抽取三元组 -> 维护 schema -> 导入 Neo4j”的主流程。
