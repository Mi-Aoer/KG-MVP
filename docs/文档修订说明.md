# 文档修改说明（一期范围收缩版）

本次修改统一将一期范围收口为：

**txt 批量导入 -> 大模型三元组抽取 -> 结果修正 -> schema 生成/维护 -> Neo4j 导入**

## 已同步处理的核心变化

### 1. 范围口径统一

- 不再把“问答”视为当前一期必做能力。
- 标题、目标、定位已从“构建与问答”收口为“图谱构建”。
- 业务闭环统一截止到“Neo4j 导入完成”。

### 2. 页面与模块统一

- 前端页面由 4 个收口为 3 个：
  - 项目与模型配置页
  - 数据导入与抽取页
  - Schema 与图谱页
- QA 页、QA Service、Cypher Validator、qa.ts、QaPage、QaResultPanel 等不再属于当前一期正式模块。

### 3. 接口与任务统一

- 当前正式接口只保留：项目、抽取配置、批次上传、抽取、重解析、三元组、Schema、图谱初始化/导入/重建。
- `T10` 问答链路已改为“后续阶段预留 / 暂缓开发”。
- `T11` 已调整为“前端 3 个最小页面”。
- `T12` 联调与验收范围已收口到“图谱导入完成”。

### 4. 数据结构口径统一

当前正式口径中，不再把问答字段和表作为一期依赖。
如为兼容旧版本需要保留以下内容，则统一按“预留、不启用、不纳入验收”处理：

- `projects.qa_config_id`
- `model_configs.config_type = qa`
- `qa_logs`

### 5. 进度记录统一

- 明确补充了范围变更原因与影响。
- 保留 `T01-T07` 已完成事实。
- 明确 `T10` 暂缓，当前主线只推进图谱构建相关能力。
- 旧记录中的问答内容统一解释为“历史范围或预留结构”，避免继续误导后续开发。
