## 本次范围变更说明

### 变更结论

自当前版本起，一期范围**暂时移除“问答”功能**。
当前一期闭环统一收口为：

`txt 批量导入 -> 大模型三元组抽取 -> 结果修正 -> schema 生成/维护 -> Neo4j 导入`

### 变更原因

原方案中的问答链路为：`用户问题 -> LLM 生成 Cypher -> 校验 -> 查询 Neo4j -> LLM 组织答案`。
考虑到该链路在问答阶段仍持续依赖大模型调用，会增加资源占用、成本和系统复杂度，因此决定在当前版本先不继续推进问答功能，等方案明确后再单独补回。

### 对当前进度口径的影响

1. `T01-T07` 已完成的事实保持不变。
2. 当前主线继续推进 `T08`、`T09`、`T11`、`T12`，目标止于“图谱构建完成”。
3. `T10` 问答链路自本次起改为“暂缓开发，不纳入当前一期”。
4. 旧记录中出现的 `qa_config_id`、`qa_logs`、`mock://qa`、问答配置、问答页等内容，统一视为：
   - 旧范围下的历史记录，或
   - 为后续扩展保留的预留结构。
   它们不再代表当前一期必做项，也不纳入当前验收口径。
